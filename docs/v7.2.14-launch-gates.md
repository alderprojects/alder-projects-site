## Post-launch decision metrics (7-14 day organic test)

After organic posts go live, track via the event log + Search Console:

- [ ] Did pages get impressions? (GSC for /guides/* and /smart-cart/topic/*)
- [ ] Did anyone click CTA? (`smart_cart_cta_click` event volume per scope)
- [ ] Did anyone start checkout? (`checkout_start` per scope)
- [ ] Did anyone purchase? (`purchase` per scope)
- [ ] Did route-out / pro clicks happen? (`route_out_click`, `contractor_lead_click`, `bridge_module_click` volumes)
- [ ] Which copy got engagement? (organic post comments — qualitative)
- [ ] Did users understand $19.99 value? (qualitative from comments)

### Decision rule

**Only build the next catalog if at least one pilot shows real pull:**

- meaningful CTA clicks (define: ≥ 30 unique CTA clicks per scope over 14 days)
- ≥ 5 checkout starts per scope
- ≥ 2 purchases per scope
- OR strong organic engagement indicating demand (sustained discussion, repeat upvotes/likes)

If neither pilot meets the bar, the right move is iteration on the existing pilots, not new catalog work. Possible iteration scopes:
- Adjust pilot catalog content based on objections from organic posts
- Tighten the bridge module copy
- Add a second authority guide for the same scope (more SEO surface)

If at least one pilot meets the bar, **next-catalog candidates** in priority order:

1. `weatherization_diy_air_sealing` (if window weatherization shows pull — same buyer)
2. `home_water_quality` re-launch with refined route-out framing (if route-out interaction is high)
3. `garage_organization` (if testing broader retail thesis — cold-start SEO)
4. `outdoor` summer scope (if summer traffic appears via lake/seasonal queries)
